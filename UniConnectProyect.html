Proyecto UniConnect

Una Aplicación Educativa Cliente-Servidor

Resumen del Proyecto

Concepto

Una aplicación de escritorio diseñada para optimizar la comunicación y la gestión de tareas entre profesores y estudiantes en un entorno de red local.

Objetivo

Crear una plataforma intuitiva y centralizada que permita a los profesores asignar actividades con recursos y a los estudiantes seguir su progreso de manera interactiva.

Tecnologías Clave

Python, Flask (para el servidor), CustomTkinter (para la interfaz), SQLite (base de datos), y Requests para la comunicación.

Arquitectura del Sistema

UniConnect opera bajo una robusta arquitectura cliente-servidor. Esta separación de responsabilidades garantiza que la lógica de negocio y los datos estén centralizados y seguros en el servidor, mientras que la interfaz de usuario en el cliente es ligera y se enfoca únicamente en la interacción.

El Servidor (Backend)

Ejecutado en un Windows Server, es el cerebro de la aplicación. Sus responsabilidades son:

  • API REST con Flask: Expone URLs (endpoints) para manejar todas las operaciones de la aplicación de forma segura.
  • Base de Datos SQLite: Almacena de forma persistente la información de usuarios, actividades, mensajes y archivos.
  • Servidor de Archivos: Gestiona la subida y descarga de materiales para las actividades en una carpeta centralizada.

El Cliente (Frontend)

Es la aplicación de escritorio que se instala en la PC de cada profesor y estudiante. Sus responsabilidades son:

  • Interfaz Gráfica con CustomTkinter: Ofrece una experiencia de usuario moderna, limpia y con un diseño profesional.
  • Interacción con el Usuario: Captura las acciones del usuario (iniciar sesión, crear actividad, enviar mensaje).
  • Comunicación con la API: Utiliza la librería 'requests' para enviar y recibir datos del servidor.

Características Principales

Pantalla de Carga Profesional

Una pantalla de inicio estilo "instalador" con barra de progreso animada que da una excelente primera impresión.

Inicio de Sesión Moderno

Una interfaz de login con un diseño pulcro, validaciones visuales y animaciones fluidas para éxito y error.

Dashboard Intuitivo por Roles

La interfaz se adapta, mostrando herramientas de gestión para el profesor y una vista de progreso para el estudiante.

Gestión de Actividades

El profesor puede crear actividades con pasos dinámicos, descripciones, imágenes y archivos adjuntos.

Seguimiento de Progreso

Los estudiantes pueden ver las actividades y marcar su progreso en un checklist interactivo.

Chat en Tiempo Real

Un chat general permite la comunicación fluida entre todos los usuarios conectados a la plataforma.

Recursos y Compilación

Recursos Utilizados

Hardware

  • Servidor: PC Genérica con procesador multi-núcleo y 4GB de RAM.
  • Cliente: Cualquier PC estándar con Windows 10 o superior.

Software

  • Sistema Operativo Servidor: Windows Server 2019.
  • Lenguaje de Programación: Python 3.13.
  • Librerías Clave: Flask, CustomTkinter, Pillow, Requests.

Compilación de la Aplicación

Para distribuir la aplicación cliente sin que los usuarios necesiten instalar Python, se utiliza la herramienta PyInstaller.

1. Instalación de PyInstaller

pip install pyinstaller

2. Comando de Compilación

Desde la terminal, en la carpeta del proyecto, se ejecuta el siguiente comando:

pyinstaller --onefile --windowed --add-data "assets;assets" client.py
  • --onefile: Empaqueta todo en un único archivo .exe.
  • --windowed: Evita que se abra una consola negra detrás de la aplicación.
  • --add-data: Incluye la carpeta de imágenes y assets en el ejecutable.
